07 — Feedback & system states

Telling residents what just happened — without drowning out availability.

The system already spends its boldness on one thing: green means available, clay means booked. Feedback colors are added carefully so they never compete with that signal. Four states, each paired with a label and icon — color is never the only cue.

Error · the one new hue

A warm brick red is the only color we add. Nothing in the palette read "stop" — so danger earns its own hue for failures and destructive actions.

Warning ≠ Clay

A yellow-gold, separated from Clay by lightness, not just hue — so it stays distinct for every color-vision type (incl. tritanopia) and in grayscale. Clay is status-only; amber is alert-only.

Success = Meadow

Success reuses the Meadow family on purpose. Green already means good/go here, so "Booked." is a green moment — no new color needed.

Info = neutral

Notices stay neutral (pine on cool paper). No foreign blue — keeping info quiet protects the green availability lane.

Tokens

Each state gets a base, an accessible ink, and a tinted surface.

Use the base for icons, dots, and solid actions; -ink for text on light backgrounds (all meet WCAG AA); -surface + -line for banner fills and borders.

Guardrails

Keep the signal clean.

The whole point of the palette is that color does real work. These rules keep the new states from muddying it.

✓ Do
  • Pair every state with an icon + text label — never color alone.
  • Use --danger red only for failures and destructive actions.
  • Let success borrow --meadow — a confirmation is a green moment.
  • Reach for -ink tokens when a state appears as text (AA).
  • Keep info neutral; it shouldn't pull attention.
✕ Don't
  • Use Clay for warnings or amber for booked — they're not interchangeable.
  • Color a spot card's availability with danger/warning hues.
  • Add a blue for info — the system has no blue on purpose.
  • Stack two solid feedback colors in one view; pick the one that matters.
  • Use red decoratively. If it's not an error, it's not red.

08 — Administrative statuses

Lifecycle states are metadata — neutral, never the availability signal.

Active, Pending, Completed, Inactive and Listed are built only from neutrals + Pine and told apart by weight + dot shape, not hue. Listed ≠ Available: the same spot is Listed (lifecycle) and Available (signal) at once — so neither can be green.

The neutral tier

Four treatments, many labels. Shape is redundant with weight — readable in grayscale.

09 — Recognition & leaderboard

The one allowed flourish: metal + Pine, never the signal hues.

Metals are always a gradient on a medallion, never a flat fill — that finish is the firewall: bronze stays distinct from flat Clay (booked), gold from flat Warning amber. Top three only; everyone else stays neutral so the podium glows by contrast.

Podium medals

Donation data stays mono/Pine — never Meadow. The champion is framed by the bay-bracket signature.
